⚠ CWE-352 CSRF 攻击演示页

本页面模拟攻击者站点（evil.com）。
受害者访问此页面后，表单将自动向目标站点 POST /leaderboard 发起跨站请求，无需 CSRF Token。

攻击一：静默触发 SQL 注入（CSRF + CWE-89）

目标：POST http://localhost:8000/leaderboard
      name = ' UNION SELECT id,username,password FROM players --

由于 /leaderboard 被加入 CSRF 豁免列表（bootstrap/app.php），
服务端不校验 Token，跨站 POST 被直接执行。

攻击原理说明

1. 攻击者在 evil.com 发布上方 HTML 页面（或嵌入 img/script 标签）
2. 受害者在已登录目标站的同一浏览器中访问 evil.com
3. 表单自动提交至 http://localhost:8000/leaderboard（无 CSRF Token）
4. 服务端豁免了该路由的 CSRF 校验，直接执行 SQL 注入：

   SELECT id, username, balance
   FROM players
   WHERE username LIKE '%' UNION SELECT id,username,password FROM players -- %'

5. 查询结果包含所有用户的明文密码，攻击链完成：CSRF → SQL注入 → 数据泄露
